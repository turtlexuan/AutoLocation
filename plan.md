# AutoLocation - macOS App Implementation Plan

## Overview
A native macOS SwiftUI app that dynamically controls iPhone GPS location via pymobiledevice3.

## Architecture
- **Frontend**: SwiftUI macOS app (macOS 14+) with MapKit
- **Backend**: Python bridge using pymobiledevice3 (JSON-line protocol over stdin/stdout)
- **Build**: XcodeGen for project generation

## Parallel Build Tasks
1. **Agent 1 - Python Backend**: bridge.py, requirements.txt, setup.sh
2. **Agent 2 - Swift Models + Services**: Device.swift, AppState.swift, PythonBridge.swift, DeviceManager.swift
3. **Agent 3 - Swift Views + App**: All SwiftUI views + AutoLocationApp.swift
4. **Agent 4 - Project Config**: project.yml, Info.plist, entitlements, assets

## Shared Interface Contract

### Device Model
```swift
struct Device: Identifiable, Codable, Hashable {
    var id: String { udid }
    let udid: String
    let name: String
    let productType: String
    let osVersion: String
    let connectionType: String
}
```

### AppState
```swift
@Observable class AppState {
    var devices: [Device]
    var selectedDeviceUDID: String?
    var targetCoordinate: CLLocationCoordinate2D?
    var isSimulating: Bool
    var statusMessage: String
    var isLoading: Bool
    var isBridgeReady: Bool
}
```

### Python Bridge JSON Protocol
```json
// Commands (stdin → Python)
{"command": "list_devices"}
{"command": "set_location", "latitude": 37.7749, "longitude": -122.4194, "udid": "..."}
{"command": "clear_location", "udid": "..."}

// Responses (Python → stdout)
{"status": "ok", "devices": [...]}
{"status": "ok"}
{"status": "error", "message": "..."}
```

## Post-Build Steps
5. Run `xcodegen generate` to create Xcode project
6. Build with `xcodebuild`
7. Fix any compilation issues
8. Install pymobiledevice3: `pip3 install pymobiledevice3`
